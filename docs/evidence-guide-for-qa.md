# Evidence Guide For QA

## Purpose

This guide helps QA and validation reviewers inspect the synthetic-data evidence
pack. It does not replace organization-specific CSV, CSA, SOP, or release
approval processes.

## Review Order

1. `reports/evidence-index.html`
2. `reports/reproducibility.json`
3. `reports/tauri-security-audit.json`
4. `reports/release-artifact-verification.json`
5. `release/validation-pack/evidence-index.json`
6. manual clean-machine checklist

## Sign-Off Boundary

The harness can persist demo review sign-off state. This is not a regulated
electronic signature. If the workflow is used for regulated electronic records
or signatures, the responsible organization must define identity, record
retention, audit trail, procedural controls, and inspection readiness.
