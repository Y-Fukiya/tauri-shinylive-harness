# Data Manager Quickstart

## Goal

Evaluate whether the synthetic data checks and review UI are useful for data
quality walkthroughs before any production EDC, SDTM, or ADaM workflow is
involved.

## Try This

1. Open `reports/clinical-data-pack-validation.json`.
2. Review required column coverage.
3. Review subject and visit references.
4. Review controlled terminology checks.
5. Review AE start/end day checks.
6. Review medication indication alignment checks.
7. Review issue summaries by subject, domain, severity, and rule.

## Evaluate

- Are query candidates understandable?
- Are visit-level references checked in a useful way?
- Are controlled terminology gaps visible?
- Are timeline checks clinically interpretable?
- Is it clear that the data are synthetic and not production data?
