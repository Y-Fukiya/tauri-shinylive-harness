# QA / Validation Quickstart

## Goal

Evaluate the evidence workflow for an offline synthetic-data demo harness.

## Try This

1. Open `reports/evidence-index.html`.
2. Review `reports/review-signoff.json`.
3. Review `reports/review-signoff-history.jsonl`.
4. Review `reports/reproducibility.json`.
5. Review `reports/tauri-security-audit.json`.
6. Review `release/validation-pack/evidence-index.json`.
7. Review manual clean-machine checklists.

## Evaluate

- Are evidence files discoverable by a non-engineer?
- Are hashes and release checksums present?
- Are missing credentials and unsigned status visible?
- Are sign-off records clearly demo evidence markers?
- Is the clinical-use limitation present in the materials being reviewed?
